# 开始

:::caution

当前版本 v0.1.x 仅支持通过 Lua 扩展，后续版本会支持更多扩展方式。

:::

## 框架扩展方式

- gRPC: 通过 `gRPC` 服务端口暴露的服务，使用支持 `gRPC` 协议的客户端调用内部函数。
- WASM: 使用框架内置的 `WASM` 运行时，执行 `WASM` 脚本调用执行库函数。
- JavaScrip: 使用框架内置的 `JavaScript` 运行时，执行 `JavaScript` 脚本调用执行库函数。
- Lua: 使用框架内置的 `Lua` 运行时，执行 `Lua` 脚本调用执行库函数。
- OneBot: 通过 `OneBot` 协议暴露的服务，使用支持 `OneBot` 协议的客户端调用执行接口。

## 扩展方式对比

| 扩展方式       | 优点                                  | 缺点                       |
|------------|-------------------------------------|--------------------------|
| gRPC       | 函数接口支持完整，可以调用几乎所有内部函数，任何语言可开发。      | 需要额外维护 `gRPC` 客户端进程。     |
| WASM       | 执行速度快，高性能，支持 `FastLib` 库，更底层的编译。    | 需要额外的 `WASM` 开发经验。       |
| JavaScript | 高级脚本语言，速度中等，高性能，支持 `FastLib` 库。     | 需要额外的 `JavaScript` 开发经验。 |
| Lua        | 简单易懂，执行速度快，高性能，快速入门，支持 `FastLib` 库。 | 无错误处理流程，直接抛出运行时报错。       |
| OneBot     | 通用协议，网络协议传输，远程连接，安全性高，分布式部署。        | 执行速度慢，需要额外维护进程。          |

## 扩展执行库

| 扩展方式       | 执行库                   |
|------------|-----------------------|
| gRPC       | `gocqhttp`, `fastlib` |
| WASM       | `fastlib`             |
| JavaScript | `fastlib`             |
| Lua        | `fastlib`             |
| OneBot     | `gocqhttp`, `fastlib` |
